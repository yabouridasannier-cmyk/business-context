## Stratégie multi-comptes

### Pourquoi multi-comptes ?
- Plus de comptes = plus d'annonces visibles = plus de ventes
- Chaque compte peut être niché différemment
- Si un compte est ban, les autres tournent encore
- Le coach a fait 15 000€ en 3 mois avec UN seul compte → multiplier par X comptes

### Setup technique

| Composant | Solution | Détail |
|---|---|---|
| Navigateur | Chrome Dev | 1 profil Chrome = 1 compte Vinted |
| Numéros téléphone | OnOff | 1 numéro virtuel par compte |
| Emails | Gmail séparés ou aliases | 1 email par compte |
| IBAN | Compte "hold" | 1 compte validé avec vrai IBAN |
| Vérification identité | Via compte hold | Bypass en achetant ses propres produits |

### Nombre de comptes testés par le coach
- **14 profils Chrome Dev sur un seul PC** — aucun problème de ban

### Risques à gérer
- Vinted détecte les comptes liés → chaque profil Chrome isole les cookies/fingerprint
- Ne PAS utiliser le même numéro ou email pour 2 comptes
- Ne PAS utiliser le même fond de photo pour 2 comptes différents

## Stratégie de niche par compte

Chaque compte = 1 niche = homogénéité = meilleur algorithme Vinted

Exemple d'architecture multi-comptes :

| Compte | Niche | Cible |
|---|---|---|
| Compte 1 | Corporate / Work femme | Chemises + pantalons larges |
| Compte 2 | Old money femme | Looks classiques, pouvoir d'achat élevé |
| Compte 3 | Streetwear femme | Casual, tendance |
| Compte 4 | Accessoires / Objets déco | Chats déco, petits objets (marge x7-x8) |

## Phases de scaling

### Phase 1 — Test (Semaine 1-2)
- 1 compte Vinted
- 10-20 annonces test
- Valider la niche + le process photo
- Commandes via marketplace (AliExpress/Temu/Shein)

### Phase 2 — Validation (Semaine 3-4)
- 2-3 comptes actifs
- 30-50 annonces par compte
- Identifier les produits best-sellers
- Commandes toujours via marketplace

### Phase 3 — Scale (Mois 2+)
- 5+ comptes actifs
- API pour poster en masse (50 annonces/heure)
- Transition vers Alibaba pour les best-sellers (si 10+ ventes/jour)
- Activation warehouse M22

### Phase 4 — Full auto (Mois 3+)
- Multi-comptes tournent en parallèle
- Warehouse gère toute la logistique
- Outil négociation automatique
- Focus = sourcing nouvelles niches + optimisation marges

## Le système "Clan M22" — décrypté

### Comment ça marche (ce qu'on ne te dit pas)
1. Tu payes M22 pour "déléguer" un compte full automatisé
2. Des élèves débutants de la formation **postulent** pour gérer les comptes à ta place
3. L'élève poste, gère le compte → prend une commission par vente
4. M22 prend une commission par vente
5. **Toi, tu prends les miettes** (le reste après toutes les commissions)

### Pourquoi on n'y va pas
- Trop d'intermédiaires = marge écrasée
- Commission élève + commission M22 + frais Vinted + frais produit + warehouse + impôts = quasi rien
- Pour que ce modèle soit rentable → il faut des niches x10+ minimum
- C'est un modèle qui profite surtout à M22 (il touche des deux côtés)

### Notre stratégie alternative
- On automatise nous-mêmes avec Claude (gratuit)
- On délègue à des agents IA, pas à des humains qui prennent des commissions
- On garde 100% de la marge
- Si un jour on doit déléguer à des humains → on le fait directement, sans passer par M22

## Réalité des marges (avec tous les frais)

### Stack de frais par vente
| Poste | Exemple (produit vendu 30€) |
|---|---|
| Commission Vinted (~5%) | -1,50€ |
| Prix achat AliExpress | -6€ |
| Warehouse (si utilisée) | -2,20€ |
| Frais envoi | -0€ (inclus Vinted) |
| **Marge avant impôts** | **~20,30€** |
| Impôts micro-entreprise (~22% URSSAF) | -4,46€ |
| **Marge nette réelle** | **~15,84€** |

→ Si tu délègues en plus via Clan M22 (commission élève + M22), ta marge tombe à ~5-8€.
→ D'où l'importance de niches x5 minimum et de NE PAS déléguer via M22.
